## What is a variable?

A variable is a name given to a value. During execution, the program accesses and optionally updates that value through this name.

JavaScript provides three keywords for declaring variables: `var`, `let`, and `const`.

<Definition term="variable">
A named reference that stores a value in memory, allowing code to read and update it by name.
</Definition>

## var, let, and const

`var` is the oldest approach; it is function-scoped and can be re-declared with the same name. This flexibility is error-prone, so modern code avoids it.

`let` is block-scoped and allows its value to be reassigned later. It is suitable for counters, loop variables, and any mutable state.

`const` is block-scoped and cannot be reassigned after its initial assignment. Use it whenever a binding should remain stable.

<CodeBlock language="javascript" title="Variable declarations">
{`var name = "Anil";         // function-scoped, can be re-declared
let counter = 0;            // block-scoped, mutable
const PI = 3.14159;         // block-scoped, cannot be reassigned

counter = 1;                // valid
// PI = 3;                  // error: Assignment to constant variable`}
</CodeBlock>

<Callout title="const and object contents">
A `const` binding to an object or array does not freeze the contents — only the reference itself cannot be reassigned.
</Callout>

## TypeScript and types

TypeScript adds static types to JavaScript. You can annotate types explicitly or let TypeScript infer them from the initial value.

<CodeBlock language="typescript" title="TypeScript type annotations">
{`let name: string = "Anil";
let age: number = 30;
let active: boolean = true;

// Type inference
let city = "Istanbul"; // TypeScript infers: string`}
</CodeBlock>

Type inference means you do not need to annotate every variable. However, explicit types are helpful in complex cases and improve readability.

<Takeaway>
Default to `const`; use `let` when mutation is necessary; avoid `var` entirely.
</Takeaway>
